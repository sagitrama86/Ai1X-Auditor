🔷 AI1X Auditor — User Guide

AI-powered HEDIS report validation tool. Version 0.1.0

Getting Started

Start the Backend

cd backend
py -m uvicorn main:app --port 8000 --reload

The backend serves the API and the UI at http://localhost:8000/ui/

Connect Your Database

Go to Connect or use the terminal:

❯ /discover

This connects to MySQL, discovers tables, and enriches them using HEDIS knowledge + Ollama.

Explore the Schema

Browse tables in Schema Explorer or the terminal:

❯ /schema
❯ /schema factqualityreport

Create a Report

Use the AI Chat terminal to define a report:

❯ Create MeasureList report with Measures, Submission, Compliance %, Denominator, Compliant

The AI generates the report definition, validates it against HEDIS rules, and saves it as YAML.

Generate SQL

❯ /sql measurelist

Generates production-grade SQL from the report YAML + field templates + joins.

Compare Results

Export data from Power BI and your SQL query, then compare on the Compare page.

Skills Library

All knowledge is stored as YAML files in the skills/ directory:

DirectoryContent

skills/domain/HEDIS foundation, technical spec, column mappings, join patterns
skills/joins/SQL JOIN definitions
skills/reports/Report definitions — fields, slicers, validations

Schema Discovery Pipeline

When you run /discover, each column is enriched in this order:

Column Mappings Cache

Check column_mappings.yaml for an exact match. Instant, no AI needed. Covers all known HEDIS RDW columns.

Ollama Enrichment

For unknown columns, Ollama generates metadata using the HEDIS tech spec + few-shot examples + sample data from the column.

Rule-Based Fallback

If Ollama is unavailable, infers metadata from column name patterns and data types.

Cache Update

New Ollama results are saved to the cache for next time.

Report Validation

The /validate command checks:

  • All template_ref values exist in field templates
  • group_by includes all non-aggregate fields
  • All 11 standard HEDIS slicers are present
  • composite_key columns exist in the field list
  • Fixed filters include isRau=0, isUtilization=0, RN=1
